# 线程的特权级切换以及上下文切换

## 🔹 一、特权级切换（Privilege‑level switch / Mode switch）

**定义来源：**

- 当用户进程发起系统调用（如 `read()` 或 `select()`）时，CPU 从用户态切换到内核态执行，这是“特权级切换”[en.wikipedia.org+11en.wikipedia.org+11en.wikipedia.org+11](https://en.wikipedia.org/wiki/System_call?utm_source=chatgpt.com)[zh.wikipedia.org+2techtarget.com+2geeksforgeeks.org+2](https://www.techtarget.com/searchsoftwarequality/tip/User-mode-vs-kernel-mode-OSes-explained?utm_source=chatgpt.com)。
- 系统调用并不涉及切换线程或进程的上下文，只是**切换权限级别**，仍然在同一线程中执行。

✅ **总结表达：**

> 特权级切换是指 CPU 在同一线程中从用户态切换到内核态（或反向），用于执行系统调用或处理中断，不涉及线程/进程上下文的保存与恢复。

## 🔹 二、线程/进程上下文切换（Thread/Process Context Switch）

**定义来源：**

- 上下文切换是**CPU 从一个线程或进程切换到另一个线程或进程**，过程包括**保存当前线程的寄存器、PC、栈等状态**，再**加载另一线程的状态**进行执行[en.wikipedia.org+8stackoverflow.com+8zh.wikipedia.org+8](https://stackoverflow.com/questions/300167/what-is-a-context-switch?utm_source=chatgpt.com)。
- Wikipedia 定义明确指出：这是“保存正在运行的进程或线程状态并且恢复另一个执行状态”的过程。

✅ **总结表达：**

> 上下文切换是指操作系统调度器将 CPU 的执行权从一个线程/进程转移到另一个，涉及状态保存与恢复，是多任务系统实现的关键机制。

✅ 区别对比表

| 对比维度   | 特权级切换（Mode Switch）                  | 线程/进程上下文切换（Context Switch）           |
| ---------- | ------------------------------------------ | ----------------------------------------------- |
| 切换对象   | 同一线程中的用户态 ↔ 内核态                | 不同线程或进程之间                              |
| CPU 执行权 | 保持原线程，切换特权级别                   | 切换 CPU 控制权到另一个线程/进程                |
| 操作涉及   | 切换运行级别，可能切换栈，但不切换线程数据 | 保存/恢复全部寄存器、程序计数器、栈指针、页表等 |
| 发生时机   | 系统调用、硬件中断                         | 抢占、多线程调度、I/O 阻塞、同步等待等          |
| 开销       | 相对较小（陷入内核 + 权限切换）            | 较大（状态保存/恢复、上下文加载/缓存失效等）    |

## 🧩 一句话精准表述

- **特权级切换**是 CPU 在同一线程中发生权限级别变化（用户态 ↔ 内核态），用于执行系统调用或中断处理。
- **上下文切换**是 CPU 从一个线程/进程切换到另一个，涉及保存和恢复完整执行状态，实现多任务并发。
